Project: Toolroom App

Concept:
- Family → Tool → Section → Part → Operations

Key rules:
- sections are fixed: N/A, Mould, Plug Assist, Anvil, Cutter, PnP, Pusher
- operations are dynamic (not fixed 6)
- some operations can be non-sequential (e.g. CAM)
- procurement = buying items
- outsourcing = external processing

Current features:
- kanban with breakdown
- suppliers database
- procurement + outsourcing integrated with suppliers
- part fields: material, quantity, size, thickness

Decisions:
- no ZIP workflow
- manual DXF handling
- email later

Next priorities:
- tool page as command center
- improve blockers and next actions
